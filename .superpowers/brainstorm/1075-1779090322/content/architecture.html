SSO Hub 整体架构

NestJS + Fastify + PostgreSQL + Traefik，全部 Docker Compose 编排

系统分层

Traefik（入口网关）
路由 · HTTPS 自动续签 · 负载均衡
SSO 认证中心
NestJS :3000
OAuth/OIDC 端点
登录页面 / 管理后台
BFF A
Fastify :3001
Token 校验
业务 A 前端页面
BFF B
Fastify :3002
Token 校验
业务 B 前端页面
PostgreSQL :5432
用户 · 应用 · Token · 审计日志

请求路径

1. 用户访问 BFF A 页面
→ BFF A 检查 session → 未登录
→ 重定向到 sso-hub.com/login
2. 用户在 SSO 登录页选择登录方式
→ GitHub OAuth / Web3 钱包签名
→ SSO 验证通过，签发 JWT token
3. 重定向回 BFF A
→ BFF A 校验 token  →  用户已登录 ✓

容器编排（6 个容器 + 2 个网络）

traefik ← 对外暴露 80/443
├── sso-hub (sso-hub.localhost)
├── bff-a (app-a.localhost)
├── bff-b (app-b.localhost)
└── postgres （内网，不对外暴露）
