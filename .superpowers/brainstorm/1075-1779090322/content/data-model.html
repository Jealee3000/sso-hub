数据模型 — ER 图

4 张核心表 + 1 张审计日志表

核心表

users（用户）
id  |  email  |  avatar_url  |  wallet_address
created_at  |  updated_at
user_identities（身份关联）
id  |  user_id → users  |  provider（github / wallet）
provider_user_id  |  provider_data（json）
created_at
oauth_clients（接入应用）
id  |  name  |  client_id（uuid） |  client_secret（hash）
redirect_uris（json） |  grants  |  scopes
is_active  |  created_at  |  updated_at
authorization_codes / tokens（授权凭证）
id  |  client_id → oauth_clients  |  user_id → users
code / token（hash） |  scopes  |  redirect_uri
expires_at  |  used（bool） |  created_at
audit_logs（审计日志）
id  |  user_id → users  |  action  |  ip_address
client_id  |  details（json）
created_at

关键关系

user ──1:N──> user_identities （一个用户可绑定 GitHub + 钱包两个身份） oauth_client ──1:N──> auth_codes / tokens （一个应用有多条授权记录） user ──1:N──> auth_codes / tokens （一个用户对多个应用授权）

设计要点

· user_identities 是多对多桥梁——email 相同的 GitHub 和钱包自动合并到同一用户
· client_secret 只存哈希，原值仅创建时返回一次
· authorization_code 单次使用后标记 used=true，防重放
· audit_logs 只追加不删改
