SSO 核心 API 端点

OAuth 2.0 + OIDC 标准端点 + 管理接口

OAuth 2.0 / OIDC 标准端点

GET /authorize
OAuth 授权入口。校验参数 → 检查登录态 → 展示登录页或直接签发 code
POST /token
用 auth_code 换 access_token + id_token + refresh_token
POST /introspect
BFF 校验 token 有效性（可选，已有 JWT 本地验签）
POST /revoke
吊销 refresh_token，用于登出
GET/POST /userinfo
OIDC 标准端点，返回当前用户身份信息
GET /.well-known/openid-configuration
OIDC Discovery 元数据，BFF 自动发现端点

登录相关端点

GET /login
登录页面（HTML），展示 GitHub / Web3 钱包按钮
GET /login/github
重定向到 GitHub OAuth 授权页
GET /login/github/callback
GitHub 回调，处理用户信息 → 签发 auth_code
POST /login/wallet/nonce
获取 Web3 签名 nonce
POST /login/wallet/verify
提交签名 → 验证 → 签发 auth_code
GET /logout
清除 Redis session → 重定向（可选 SLO 广播）

管理后台 API（需管理员 session）

CRUD /admin/clients
管理 OAuth 应用注册
CRUD /admin/users
用户管理（列表、禁用、删除）
GET /admin/audit-logs
审计日志查询（按用户、应用、时间筛选）

JWT 令牌设计

access_token（JWT，15 min）
{ sub, client_id, scopes, iat, exp }
BFF 本地验签，无需每次调 SSO
id_token（JWT，15 min）
{ sub, email?, name?, avatar_url?, iat, exp }
OIDC 用户身份信息，BFF 解码展示
refresh_token（opaque，7 天）
存储在 Redis + PG 双写，Redis 快查，PG 持久保底
