登录流程

GitHub OAuth + Web3 钱包，统一走 OAuth 2.0 授权码流程

整体时序

用户访问 app-a.com → BFF A 检查 session → 无 session
→ 302 重定向到 sso-hub.com/authorize?client_id=bff-a&redirect_uri=app-a.com/callback
SSO 显示登录页面，用户选择 Github 或 钱包：
GitHub OAuth 分支：
SSO → 302 → github.com/login/oauth/authorize
用户授权 → github 回调 SSO → SSO 拿到 GitHub user info
SSO 查找/创建 user → 创建 user_identity(provider=github)
Web3 钱包分支：
SSO 生成 nonce → 用户钱包签名 → SSO 验签
SSO 查找/创建 user → 创建 user_identity(provider=wallet)
两个分支汇合：
SSO 生成 authorization_code → 302 重定向到 app-a.com/callback?code=xxx
BFF A 收到 code：
服务端 POST sso-hub.com/token { code, client_id, client_secret }
SSO 验证 → 返回 access_token + id_token（JWT）
BFF 存 JWT 到 session → 浏览器 set-cookie: session_id=xxx
后续请求：
浏览器带 cookie → BFF 从 session 取出 JWT → 渲染页面

关键安全点

· authorization_code 一次性使用，用完即标记 used
· authorization_code 5 分钟过期，超时作废
· redirect_uri 必须和注册时完全匹配，防重定向劫持
· /token 端点只接受服务端调用，PKCE 可选（BFF 是 confidential client）
· nonce 签名验证保证 钱包私钥从未离开用户设备
· session httpOnly + Secure + SameSite=Lax
